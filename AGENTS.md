# Repository instructions

## Commit-message handoff

Whenever you modify, add, rename, or delete files in this repository, include a suggested commit message in the final response.

- Format it as: `Suggested commit message: <message>`
- Use a concise, imperative message that describes the completed change.
- Prefer a Conventional Commit prefix such as `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, or `chore:`.
- Keep the subject line at 72 characters or fewer when practical.
- Do not create a Git commit unless the user explicitly asks you to commit.
- If no repository files changed, do not include a suggested commit message.
